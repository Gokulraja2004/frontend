AWS TASK: VPC Infrastructure

Architecture: Public & Private Subnet | NAT Gateway | Bastion Host | Nginx | ALB

1. Project Overview

This project demonstrates a complete AWS networking and web hosting setup. The goal is to securely host a web application inside a private subnet while making it accessible through a public Application Load Balancer (ALB).

Technologies: AWS VPC EC2 NAT Gateway Nginx ALB Linux

2. Network Configuration

VPC & Subnet Details

Resource Name CIDR Block
VPC demo-vpc 10.0.0.0/16
Public Subnet pu-sub 10.0.1.0/24
Private Subnet pri-sub 10.0.2.0/24

Routing & Gateways

  • Internet Gateway: Attached to VPC to provide public internet access.
  • NAT Gateway: Deployed in the public subnet to allow private instances to download updates.
  • Public Route Table: Routes 0.0.0.0/0 to the Internet Gateway.
  • Private Route Table: Routes 0.0.0.0/0 to the NAT Gateway.

3. Server Deployment

EC2 Instances

  • Bastion Host (Public): Acts as a secure jump server for SSH management.
  • Private EC2 (Nginx): Hosts the Calculator and Portfolio securely without a public IP.

Web Server Setup

Commands used to configure the Nginx server in the private instance:

sudo yum install nginx -y
sudo systemctl enable nginx
sudo systemctl start nginx

4. Security Group Rules

Security Group Port Source / Allowed Traffic
Bastion SG 22 (SSH) Administrator IP (My IP)
Private EC2 SG 80 (HTTP) ALB Security Group
Private EC2 SG 22 (SSH) Bastion Security Group

5. Load Balancer & Final Access

An Application Load Balancer (ALB) was configured to manage incoming web traffic:

  • Target Group: Private EC2 instance registered on Port 80.
  • Health Check: Verified status as 'Healthy' for path "/".

Application URL: http://my-load-balancer-517638320.ap-south-1.elb.amazonaws.com

  • Implemented multi-tier architecture with Public/Private isolation.

  • Configured NAT Gateways for secure outbound-only internet connectivity.
